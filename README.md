# Решение ООО"Нтон" — Система автоматической классификации почты

> Конвеерная система сортировки писем система для IT-отделов: автоматически читает входящие письма, сортирует их по смысловым категориям и формирует подробную статистику обработки.

## О проекте

Отделы IT-поддержки ежедневно получают сотни писем — критические инциденты, рутинные запросы, спам. Ручная сортировка отнимает время и неизбежно приводит к потерям важных обращений.

**Система от ООО"Нтон"** решает эту проблему: читает входящие письма из файловой системы, классифицирует их по категориям (критические инциденты, запросы, спам и др.) и раскладывает по соответствующим папкам.

### Ключевые возможности

- **Чтение писем** из структуры директорий, имитирующей почтовый ящик
- **Автоматическая классификация** по смысловым категориям
- **Перемещение файлов** в соответствующие папки по результатам классификации
- **Лог и статистика** — детальный отчёт о каждом запуске
- **Два режима работы** — разовое выполнение и непрерывный серверный режим
- **Устойчивость к неизвестным категориям** — система корректно обрабатывает письма, не подпадающие ни под одну из заданных категорий

## Запуск

В проекте предусмотрено два bash-скрипта запуска, расположенных в директории `scripts/`.

### Вариант 1 — Разовое выполнение (`basic.sh`)

Запускает классификатор **один раз**: обрабатывает все письма во входящих и завершает работу. Подходит для ручного запуска, отладки и задач по расписанию (cron).

#### Шаги запуска

**1. Клонируйте репозиторий и перейдите в корень проекта:**

```bash
git clone https://github.com/svamst/OOOnton_tp_hackathon.git
cd OOOnton_tp_hackathon
```

**2. Дайте скрипту права на выполнение:**

```bash
chmod +x scripts/basic.sh
```

**3. Запустите скрипт из корня проекта:**

```bash
./scripts/basic.sh
```

#### Что происходит при запуске

```
[01-06-2025 12:00:00] Классификатор писем запущен. Отслеживается процесс: /путь/к/src/main.py
[01-06-2025 12:00:00] Запуск /путь/к/src/main.py
...
[01-06-2025 12:00:05] Программа завершилась успешно. Время работы: 5 секунд
```

- Лог сохраняется в `logs/lifetime/run.log`
- При успешном завершении выводится время работы
- При ошибке — код завершения и время работы

---

### Вариант 2 — Серверный режим (`server.sh`)

Запускает классификатор в **бесконечном цикле**: после каждого завершения (успешного или с ошибкой) программа автоматически перезапускается через 5 секунд. Предназначен для работы на сервере как фоновый сервис.

#### Шаги запуска

**1. Клонируйте репозиторий и перейдите в корень проекта:**

```bash
git clone https://github.com/svamst/OOOnton_tp_hackathon.git
cd OOOnton_tp_hackathon
```

**2. Дайте скрипту права на выполнение:**

```bash
chmod +x scripts/server.sh
```

**3. Запустите скрипт в фоновом режиме:**

```bash
nohup ./scripts/server.sh &
```

> `nohup` гарантирует, что процесс продолжит работу после закрытия терминала или завершения SSH-сессии.

**Либо в текущем терминале (для отладки):**

```bash
./scripts/server.sh
```

#### Что происходит при запуске

```
[01-06-2025 12:00:00] Классификатор писем запущен. Отслеживается процесс: /путь/к/src/main.py
[01-06-2025 12:00:00] Запуск /путь/к/src/main.py
...
[01-06-2025 12:00:05] Программа завершилась успешно. Время работы: 5 секунд
[01-06-2025 12:00:05] Перезапуск через 2 секунды...
[01-06-2025 12:00:10] Запуск /путь/к/src/main.py
...
```

#### Остановка серверного режима

Если скрипт запущен через `nohup`:

```bash
# Найти PID процесса
pgrep -f server.sh

# Завершить процесс
kill <PID>
```

Если скрипт запущен в текущем терминале — нажмите `Ctrl+C`.

---

### Сравнение режимов запуска

| | `basic.sh` | `server.sh` |
|---|---|---|
| **Режим работы** | Однократный запуск | Непрерывный цикл |
| **Перезапуск при ошибке** | ❌ | ✅ (через 5 сек) |
| **Перезапуск после успеха** | ❌ | ✅ (через 2 сек) |
| **Когда использовать** | Разработка, cron, ручной запуск | Продакшн-сервер |
| **Логирование** | `logs/lifetime/run.log` | `logs/lifetime/run.log` |

### Ключевые компоненты системы
1. Парсинг и устойчивость данных (EmailParser)
Многоуровневое декодирование: Поддерживает автоматический подбор кодировок (utf-8, utf-8-sig, cp1251, latin-1) для исключения падения системы при чтении писем из старых почтовых клиентов.

Фильтрация бинарного шума: Файлы изображений, архивов и исполняемых файлов (.png, .zip, .exe и др.) детектируются на этапе анализа расширения и изолируются.

Парсер одинаково успешно обрабатывает как структурированные файлы JSON (извлекая поля subject и body), так и сырой текст с использованием регулярных выражений

2. Бизнес-логика классификации (RulesClassifier)
Распределение контента происходит по четырем базовым категориям:

critical_incidents — критические сбои, падение систем (ошибки 500, недоступность шлюзов).
spam_and_phishing — рекламные рассылки, фишинговые атаки, подозрительные ссылки.
routine_requests — стандартные запросы (доступы, оборудование, отпуска).
outliers — поврежденные файлы, пустые документы и бинарные данные.

Все текстовые паттерны приводятся к нижнему регистру для обеспечения регистронезависимого поиска.

3. Файловый менеджер и безопасность (FileManager)
Идемпотентность структуры: При старте система проверяет наличие необходимых целевых папок и создает их в случае отсутствия.

Разрешение коллизий: В случае совпадения имен файлов в целевой директории алгоритм автоматически генерирует уникальный числовой суффикс (например, mail_1.txt), предотвращая затирание или перезапись обработанных данных.

### Тестирование

> Проект покрыт модульными тестами с использованием фреймворка pytest. Тесты изолированы от конфигурационных файлов благодаря механизму monkeypatch, проверяют краевые случаи (пустые файлы, капслок, падение парсера) и гарантируют стабильность классификации контента.

Запуск тестового пакета:

Bash
pytest
